# Crucible

The **Crucible** converts stone-like inputs into lava using heat from the block below.

## Core behavior

- Stores both queued solids and generated lava in block states.
- Produces lava in steps of **250 mB**.
- Requires heat source under the block to progress smelting.

## Accepted inputs

Default conversion units in runtime:

- Cobblestone / Stone / Diorite / Granite / Blackstone: `1`
- Netherrack: `4`

> Source: `acceptedItems` in `BP/scripts/blocks/crucible.js`

## Heat progression

Heat source strength is additive per tick cycle and smelt completes when progress passes threshold.

> Source: `heatSources` + `onTick` in `BP/scripts/blocks/crucible.js`

## Capacity and safety rules

- Lava state max: `4` (1,000 mB total)
- Internal insertion now respects remaining capacity before accepting new solid units.
- This prevents overflow edge cases when using high-value inputs (like netherrack).

## Fluid extraction compatibility

Fluid extractors can pull Crucible lava.

Important runtime behavior:

- transfer is quantized to **250 mB steps**
- drain is proportional to actual transferred amount

> Source: `BP/scripts/machinery/transfer_system/system.js`
