## Network Model (Script)

- Item networks are discovered by an automatic scan when the system updates.
- Color channels let you run separate item lines in parallel.
- Exporters cache discovered targets and rebuild that cache when the network changes.
- Importers are resolved by world position so exporters can route items correctly.

---

## Exporter Behavior

- Source container = the block directly in front of the exporter.
- Transfer mode:
  - nearest
  - farthest
  - round-robin
- Tick interval by speed tier:
  - 0 = 10t
  - 1 = 8t
  - 2 = 6t
  - 3 = 4t
  - 4 = 2t
- Runtime send cap is 64 items per cycle.
- Exporter menu supports:
  - ON/OFF toggle
  - transfer mode selection
  - whitelist/blacklist toggle
  - add/remove filter entries (requires filter upgrade)

---

## Importer Behavior

- Importer does not spawn its own pipe entity.
- Importer settings are stored per block position.
- Supports:
  - normal filter (whitelist / blacklist)
  - smart importer mode with slot mapping
- Exporters resolve the real destination container in front of importer face before item insertion.

---

## Quick Setup Pattern

1. Place a source container/machine.
2. Face an <b>Item Exporter</b> toward the source.
3. Connect paths with <b>Item Conduit</b>.
4. Place <b>Item Importer</b> facing the destination.
5. Configure mode/filter only where needed.
